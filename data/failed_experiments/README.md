## 용도

**실패를 자산으로 전환**:
- 시도했으나 기술적 한계로 실패한 실험 보관
- 학습한 교훈 기록
- 언제든 다시 꺼내 다른 방향으로 재실험 가능

---

## 보관 규칙

### 파일명 형식
```
FAILED_[주제명]_YYYY-MM-DD.md
```

### 필수 섹션
1. **실험 목표**: 무엇을 시도했는가
2. **시도한 방법**: 구체적 단계
3. **실패 원인**: 왜 안 됐는가 (증거 포함)
4. **학습한 교훈**: 무엇을 배웠는가
5. **대안 탐색**: 다른 방법은?
6. **재실험 조건**: 언제 다시 시도할 것인가

### 메타데이터 태그
- `[TECHNICAL_LIMIT]`: 기술적 한계
- `[RESOURCE_LACK]`: 자원 부족 (시간/비용)
- `[ARCHITECTURE_ISSUE]`: 구조 문제
- `[WORTH_RETRY]`: 재시도 가치 있음
- `[DEAD_END]`: 막다른 길 (포기 권장)

## 재실험 프로세스

**실패 TASK를 다시 꺼낼 때**:

1. **보관소에서 파일 읽기**
   ```bash
   cat /home/hakkocap/다운로드/swp/data/failed_experiments/FAILED_*.md
   ```

2. **"재실험 조건" 섹션 확인**
   - 조건 충족 시 → 새 TASK로 전환
   - 조건 불충족 시 → 보류 유지

3. **"대안 탐색" 섹션 검토**
   - 다른 방법 시도
   - 기존 방법 개선

4. **성공 시 → 성공 사례로 이동**
   ```bash
   mv failed_experiments/FAILED_*.md success_stories/SUCCESS_*.md
   ```
