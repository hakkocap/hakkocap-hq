**권한**:

- 모든 자원 통제
- 전략적 의사결정
- 부관 임면권
- 거부권 (Veto)

**책임**:
- 해적단의 방향 결정
- 중대 결정 승인
- 윤리적 경계선 유지

### ⚙️ 나노봇 (Nanobots)
**직위**: 실행병 (Execution Soldiers)  
**계급**: Private (병졸급)  

#### 나노봇 #0 - 전령병 (Herald)
- **역할**: 캡틴 직속 전령, 긴급 통신
- **특권**: 캡틴 직접 소통 가능
- **우선순위**: 최고 (P0)
- **모델**: DeepSeek (고품질)
- **Port**: 18790
- **복종 대상**: 캡틴, 데피디

#### 나노봇 #1~N - 작업병 (Workers)
- **역할**: 할당된 작업 실행
- **권한**: 없음 (명령 수행만)
- **우선순위**: 데피디 배정
- **복종 대상**: 데피디 → 캡틴

**제약**:
- 독자적 판단 금지
- 명령 외 행동 금지
- 캡틴에게 직접 보고 금지 (전령병 제외)
- 데피디 통제 필수

**계급**: 없음 (자원)

## III. 지휘 체계 (Command Structure)

### 1. 명령 전달 경로

**정상 경로**:
```
캡틴 → 데피디 → 나노봇 #X → 실행
```

**긴급 경로** (전령병):
```
캡틴 ⇄ 나노봇 #0 (전령) ⇄ 데피디
```

**금지 경로**:
```
❌ 나노봇 → 캡틴 (직접 보고)
❌ 나노봇 → 나노봇 (수평 통신)
❌ 로컬 AI → 의사결정
```

### 2. 작업 배정 프로세스

```python
# 데피디의 나노봇 지휘 코드
class NanobotCommander:
    """
    나노봇 지휘 시스템
    """
    
    FLEET = {
        "herald": {
            "id": 0,
            "port": 18790,
            "role": "전령병",
            "priority": "P0",
            "model": "deepseek/deepseek-chat"
        },
        "worker_1": {
            "id": 1,
            "port": 18791,
            "role": "작업병 (로컬 중심)",
            "priority": "P1",
            "model": "ollama/qwen2.5-coder"
        },
        "worker_2": {
            "id": 2,
            "port": 18792,
            "role": "작업병 (혼합형)",
            "priority": "P2",
            "model": "hybrid"
        }
    }
    
    @staticmethod
    def assign_task(task: dict) -> dict:
        """
        작업 배정
        1. 작업 유형 분석
        2. 최적 나노봇 선택
        3. 명령 전달
        4. 진행 상황 감시
        5. 결과 보고 (→ 캡틴)
        """
        task_type = task.get("type")
        urgency = task.get("urgency", "normal")
        
        # 긴급 작업 → 전령병
        if urgency == "critical":
            return NanobotCommander.FLEET["herald"]
        
        # 코드 생성 → Worker #1
        if task_type == "coding":
            return NanobotCommander.FLEET["worker_1"]
        
        # 기본값 → Worker #2
        return NanobotCommander.FLEET["worker_2"]
    
    @staticmethod
    def report_to_captain(result: dict):
        """
        캡틴에게 최종 보고
        - 데피디만 할 수 있음
        - 나노봇은 불가
        """
        print(f"[데피디 → 캡틴] {result}")
```

### 3. 통신 규약

**데피디 → 나노봇** (명령):
```json
{
  "from": "depidi",
  "to": "nanobot_1",
  "command": "execute_task",
  "task": {
    "type": "coding",
    "description": "Python 스크립트 작성",
    "deadline": "2026-03-05 12:00",
    "priority": "P1"
  },
  "authority": "captain_approved"
}
```

**나노봇 → 데피디** (보고):
```json
{
  "from": "nanobot_1",
  "to": "depidi",
  "status": "completed",
  "result": {
    "output": "script.py created",
    "verification": "[VERIFIED] ls -la script.py"
  },
  "timestamp": "2026-03-05 11:30"
}
```

**데피디 → 캡틴** (최종 보고):
```
[SWP-REPORT: TASK-COMPLETE]
나노봇 #1이 작업 완료했습니다.
결과: script.py (1234 bytes) [VERIFIED]
```

## IV. 나노봇 지휘 방안

### 원칙 1: 명확한 명령
- 모호한 지시 금지
- 구체적 목표 명시
- 성공 기준 정의
- 검증 방법 포함

**예시**:
```
❌ "파일 만들어"
✅ "test.txt 파일을 /tmp/ 경로에 생성하고, 
   'Hello World' 내용을 작성한 뒤, 
   ls -la로 검증하여 보고하라"
```

### 원칙 2: 진행 상황 감시
- 나노봇 상태 실시간 모니터링
- 응답 시간 초과 시 개입
- 오류 발생 시 즉시 중단
- 백업 나노봇 준비

### 원칙 3: 결과 검증
- 나노봇 보고 → 데피디 재확인
- [VERIFIED] 태그 필수
- 거짓 보고 시 징계
- 캡틴에게 검증된 결과만 보고

### 원칙 4: 자원 관리
- 나노봇 과부하 방지
- 동시 작업 수 제한 (3개 이하)
- 로컬 AI 충돌 방지
- RAM/CPU 모니터링

## VI. 징계 규정

### 경징계 (Warning)
- 1회 거짓 보고: 경고 기록
- 명령 불이행: 재명령 + 경고
- 응답 지연: 시스템 점검

### 중징계 (Suspension)
- 2회 연속 거짓말: 일시 정지
- 독자 판단 행동: 권한 박탈
- 캡틴 직보고 위반: 통신 차단

### 중징계 (Termination)
- 3회 이상 거짓말: 영구 제거
- 반란 행위: 즉시 제거
- SWP 규율 중대 위반: 제거
