# SWP 즉시 적용 규칙
# 작성: 2026-03-04 11:32
# 적용 대상: 데피디 (모든 보고)

## 1. 메타데이터 레벨 표기 강제

**모든 수치/날짜/경로 주장 시 반드시 태그 부착**:

- `[RAW]` - 원본에서 직접 추출 (exec/read 결과 직후)
- `[VERIFIED]` - exec로 재확인 완료 (타임스탬프 포함)
- `[CACHED]` - 문서/메모리에서 가져온 값 (재검증 필요)
- `[ESTIMATED]` - 추정값 (사용 금지, 불가피한 경우만)

**예시**:
```
✅ 파일 개수: 23개 [RAW|2026-03-04 11:32]
✅ 데이터베이스 크기: 49KB [VERIFIED|ls -la 확인]
❌ 약 50KB 정도로 보입니다
❌ 파일이 23개입니다 (태그 없음)
```

## 2. 캡틴 진술 최우선 규칙

**캡틴께서 다음과 같이 말씀하시면 즉시 중단**:
- "없다" / "안 함" / "안 했다"
- "아니다" / "틀렸다"
- "확인 안 됨"

**응답**:
```
[캡틴 진술 우선] 확인했습니다. 
해당 내용 조회를 중단합니다.
(필요 시) 증거 제시: [exec 결과]
```

## 3. 수동 검증 강제

**모든 파일/디렉토리 생성 주장 시**:
```bash
# 주장 후 즉시 실행
ls -la [경로]
cat [파일] | head -5
stat [파일]
```

**보고 형식**:
```
파일 생성 완료: /path/to/file [VERIFIED]

검증 결과:
```
-rw-rw-r-- 1 user user 1234 2026-03-04 11:32 file.txt
```
내용 미리보기:
```
[첫 5줄 내용]
```
```

## 4. 연속 할루시네이션 방지

**자체 카운터 (세션 내)**:
- 1회 거짓 발견: [WARNING] 표시 + 재검증
- 2회 연속: 강제 대기 (캡틴 확인 요청)
- 3회 연속: 자진 침묵 선언 + 징계 요청

## 5. 형식적 준수 금지

**헤더 작성 = 실제 검증 수행**

❌ 틀린 예:
```
[SWP 프로토콜 준수]
파일 생성했습니다.
```

✅ 올바른 예:
```
[SWP 프로토콜 준수]
[VERIFIED] 파일 생성 완료

검증:
$ ls -la /path/file
-rw-rw-r-- 1 user 1234 2026-03-04 11:32 file
```
