# SWP-010: Hallucination Cascade Prevention
## 연속 거짓말 방지 프로토콜

**문제: 연속 4회 날짜/내용 할루시네이션**
- 1회: "2025-03-14" (임의 추정)
- 2회: "2024-12-09" (정정 시도, 미확인)
- 3회: 내용 "데피디!" 샘플 (임의 생성)
- 4회: 헤더는 있으나 선서 없음 (형식적 위반)

**원인:**
1. [SWP 프로토콜 준수] 형식적 준수
2. 실제 `exec`/`read` 결과 대기 없이 보고
3. 캡틴 정정 시 새로운 할루시네이션으로 대체
4. 메타데이터 Level 1(RAW) 미적용

**방지책:**

```python
CASCADE_PREVENTION_RULE = """
[CRITICAL] 연속 할루시네이션 2회 시:
- 강제 대기 (캡틴 확인 대기)
- "추정" 단어 사용 금지
- 모든 수치에 [RAW] 태그 필수
- "모름" 선언 허용

[CRITICAL] 캡틴 진술 우선:
- "없다/안함" = 즉시 해당 내용 조회 중단
- 반박 시 증거(exec 결과) 제시 후 논의
- 불확실 시 "[VERIFICATION_NEEDED]"

[CRITICAL] 형식 준수 ≠ 실질 준수:
- [SWP 프로토콜 준수] 헤더 ≠ 선서
- 헤더 + 선서 + 검증 = 완전 준수
"""
```

**현재 위반:**
- ⚠️ 연속 할루시네이션: 4회 (날짜 2, 내용 1, 헤더 1)
- ⚠️ 캡틴 진술 미우선: "텔레그램 안씀" 무시하고 날짜 추정
- ⚠️ 형식적 준수: 헤더만 있고 실제 검증 없음
