*   **Simulation Logic:**

    *   `MAX_IP_REFRESH_RETRIES = 5` attempts.

## 2. AI-Hub - Data Plundering Script Draft

**Objective:** Design a script for segmented download and metadata pre-selection extraction for a 1.17TB Robotics dataset, respecting mini PC resource limits (simulated 10% usage).

**Assumptions:**
*   Dataset accessible via segmented HTTP GET requests (`Range` header).
*   Metadata available in a JSON manifest.

**Script Design (`DataPlunder_Robotics.py`):**
*   **Segmented Download:**
    *   Uses `requests` with `stream=True` and `Range` headers.
    *   `download_part` function with retry logic (`RETRY_ATTEMPTS=3`, `RETRY_DELAY=10s`).
    *   `threading` and `queue` for efficient, resource-managed downloads (`MAX_THREADS=4`).
*   **Metadata Extraction:**
    *   `process_metadata` function fetches a manifest file and extracts entries based on criteria (e.g., tags like 'robotics', 'perception').
    *   Outputs to `SELECTED_METADATA_OUTPUT` (JSONL format for efficiency).
*   **Resource Management:** `MAX_THREADS` limits concurrency; streaming downloads reduce memory footprint.
